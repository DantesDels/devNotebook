Commentaires

Bonnes pratiques

  • Commentez le "pourquoi" plutôt que le "quoi".
  • Utilisez Javadoc pour les API publiques.
  • Gardez les commentaires synchronisés avec le code.
  • Documentez les effets de bord et préconditions.

À éviter

  • Commentaires évidents ou redondants avec le code.
  • Code commenté mort: supprimez-le ou utilisez le contrôle de version.
